2. Declare your object in the header file

Your objects should inherit from the SimObject class. But headsup when you‚Äôre writing constructor function for your object: The constructor for all SimObjects assumes it will take a parameter object. This parameter is automatically created by the build from the python class for the simobject. The name for this parameter type is generated automatically from the name of your object. For our ‚ÄúHelloObject‚Äù the parameter type‚Äôs name is ‚ÄúHelloObjectParams‚Äù. To use this type, include params/HelloObject.hh or replace the header file name with yours.

Also, to avoid compiling errors, in gem5 you should

  • wrap all header files in ifndef/endif with the name of the file and the dir it‚Äôs at.
  • declare Simobjects within the gem5 namespace scope.

#ifndef checks whether the given token has been #define-d earlier in the file or in an included file; if not, it includes the code between it and the closing #else or, if no #else is present, #endif statement. #ifndef is often used to make header files idempotent by defining a token once the file has been included and checking that the token was not set at the top of that file.
